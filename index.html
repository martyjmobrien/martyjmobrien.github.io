Still Alive...

2nd Feb 2017

After a very long period away from coding I am (finally) back.

Previously I took the path of Codecademy, Learn to Program etc and whilst they were all great resources (bar some hand holding) I am taking a different approach now. What I felt was lacking previously was a clear path. One which I could work through and one where each step fed into the next to build and reinforce knowledge. With this in mind I am working my way through The Odin Project. It is still early days but I really like how it is a) structured and b) geared around using the best of what the internet has to offer and not just giving one perspective.

An example of this was in the introductory lessons on using git. Not only did it talk about what git is, the difference between git and GitHub and the basics of how to use it in the terminal but it also linked to outside resources which reinforced these ideas, gave different/more thorough explanations and also went into more detailed terminology. I also find that creating a repository, cloning it to your computer, seeing the files appear and seeing realtime changes when pushing to GitHub gives a much greater understanding compared to Codecademy (which the Odin project actually links to as a very basic initial lesson in git).

At the same time as this I am also still looking at the Duplicate File finding app I was making. When I left it it could only add an incrementing number to the very end of the file (after the extension) and the numbers wouldn’t be specific to the file dupe count, instead it would just keep counting from 1 (so eight varying dupes would result in the 8th dupe having the number eight, even if there was one duplicate of that particular file). Having come back to it I realised two things:

1: The use of File.basename was only removing the file path - I wasn’t giving it an argument to remove the extension as well.

2: Changing the basename_array to a hash would allow me to track individual file duplications by incrementing their value.

By implementing these changes I was able to hold the ext name in a variable, strip just the file name out, check it against the hash, add the value to the end of the file, increment the value and then add the original ext name back on. Phew!

Still more to do (such as specifying where the files are/where you want the duplicates to be saved) by it’s always nice to overcome a challenge, however small!

I Haven't Given Up!

26th May 2016

It may seem as though I have thrown in the coding towel and called it quits but really I’ve just been lazy at keeping up with posts.

In the time between my last post and this one I have completed the codecademy HTML and CSS courses as well as the Ruby course. I have now moved on to learning the UNIX terminal and More Ruby using Zed Shaw’s Learn Ruby The Hard Way. The main reason I started learning Ruby from Zed Shaw after completing the Codecademy course was because I felt like I didn’t really finish the course with the knowledge I wanted or thought I should have attained. My biggest problem was using the browser based code editor. I felt this created a disconnect by simply allowing you to run code and instantly see the results. As a coding newbie I really value the approach in Learn Ruby The Hard Way as you have to write out the code in an editor (Wrangler for me) and then use the terminal to run the code. This has made me feel more connected to the process and has also allowed me to further my knowledge of the command line.

I have found learning more about the terminal and it’s commands really interesting. The ability to change in and out of folders, create new documents, list contents etc has given me a much greater sense of what is actually happening ‘under the hood’, so to speak. Unlike simply double clicking or drag and drop, the terminal has a more tactile feel to it. Having to actually tell the computer “OK, make a new folder, move everything from that other folder into the new one and then delete the old one” gives an actual sense of achievement, which for someone new to coding is really important. The little victories really make learning fun and interesting, at least for myself. Take this simple terminal command:

echo This is my new file > new_file.txt

Having that command actually produce a new file with “This is my new file” as it’s content made me want to learn and try more as it’s a tangible product of what I have learnt.

I am only half way through LRTHW but at this point I am not only enjoying it more than Codecademy but I also feel that I am learning more. One of the things I love is that it is challenging and Zed doesn’t try to hide that fact in any way. I will update on my experience as a whole once I finish the course but at this point I would have to say that LRTHW is a better experience. The code isn’t hidden and there isn’t any hand holding.

I will try to be better at updating my learning experience from here on out which will hopefully mean less of these mammoth entries!

And So It Begins.

14th Apr 2016

I have just completed the Stanford CS101 online course over at Coursera and have now moved on to the HTML & CSS course at Codecademy. These are my impressions of both so far.

CS101:

The stanford CS101 course is a very worthwhile course to go through for a beginner like myself. It not only introduces you to coding but gives insight into the mindset of a programmer. Nick Parlante, the course lecturer, is superb at what he does. Engaging, interesting, and highly intelligent.

Codecademy HTML & CSS:

Overall I would say this course is good so far but there are certain areas in which I feel quite frustrated. Often times you will get to the end of a section and it will say “Why not try adding images in the divs you have just made” just for you to spend ages banging your head against a wall until you find out that ‘Position in CSS’ is the next course?! Also, in some parts I have found it to be too hand holding, telling you parameters to add in their code form instead of simply stating what is needed and making the user think and remember.

I am near the end of the CSS course now and I think that I will go through the whole HTML/CSS course again to make sure it has all gone in. One tactic I have been using, which I think has helped, has been to use a HTML/CSS editor (in my case Brackets) along the Codecademy course. By doing so I have been able to explore what changing certain pieces of coding does etc without the restraints of the inbuilt Codecademy editor telling me I haven’t successfully inputted the course EXACTLY as it want me too.
